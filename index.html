Diagramas Entidad-Relación

Introducción básica al diseño de bases de datos

1. ¿Qué es un diagrama Entidad-Relación?

Un diagrama Entidad-Relación, también llamado diagrama E/R, es un dibujo que nos ayuda a organizar la información antes de crear una base de datos.

Idea principal: antes de crear tablas en una base de datos, necesitamos pensar qué información queremos guardar, cómo se agrupa y cómo se relaciona entre sí.

2. Elementos básicos de un diagrama E/R

Entidad

Es algo importante de lo que queremos guardar información.

Ejemplos: Cliente, Libro, Alumno, Producto.

Atributo o campo

Es una característica o dato de una entidad.

Ejemplos: nombre, teléfono, precio, fecha.

Campo clave

Es el campo que identifica de forma única a cada registro.

Ejemplo: id_cliente.

Relación

Indica cómo se conectan dos entidades.

Ejemplos: compra, presta, pertenece, realiza.

3. Cómo se ponen los campos y el campo clave

En un diagrama sencillo podemos dibujar cada entidad como una caja. Arriba ponemos el nombre de la entidad y debajo escribimos sus campos.

El campo clave se suele marcar subrayado o con la indicación PK, que significa Primary Key o clave primaria.

PK: clave primaria, identifica cada registro. FK: clave ajena, conecta con otra tabla. Subrayado: también puede indicar campo clave.
Ejemplo: dos clientes pueden llamarse igual, pero no pueden tener el mismo id_cliente. Por eso id_cliente es el campo clave.

4. Cardinalidades básicas

Las cardinalidades indican cuántos elementos de una entidad pueden relacionarse con elementos de otra entidad.

Cardinalidad Significado Ejemplo
1:1 Un elemento se relaciona con un único elemento. Una persona tiene un DNI.
1:N Un elemento se relaciona con muchos elementos. Un profesor imparte muchas asignaturas.
N:M Muchos elementos se relacionan con muchos elementos. Un alumno puede matricularse en varias asignaturas y una asignatura puede tener varios alumnos.

5. Problema de ejemplo: tienda de videojuegos

Enunciado del cliente

Una pequeña tienda de videojuegos llamada GameZone necesita crear una base de datos para organizar mejor la información de su negocio.

Actualmente apuntan las ventas en papel y están teniendo problemas para saber qué clientes compran más, qué videojuegos tienen disponibles y qué juegos ha comprado cada cliente.

La tienda necesita guardar la siguiente información:

  • De cada cliente quiere guardar su nombre y teléfono de contacto.
  • De cada videojuego quiere guardar el título, la plataforma y el precio.
  • De cada compra quiere guardar qué cliente realizó la compra, qué videojuego compró y la fecha de la compra.

6. Solución del problema

Paso 1: Identificar las entidades

  • CLIENTE
  • VIDEOJUEGO
  • COMPRA

Paso 2: Identificar los atributos y las claves

Entidad Campos Campo clave
CLIENTE id_cliente, nombre, telefono id_cliente
VIDEOJUEGO id_juego, titulo, plataforma, precio id_juego
COMPRA id_compra, fecha_compra, id_cliente, id_juego id_compra
En la entidad COMPRA aparecen también id_cliente e id_juego. Estos campos sirven para saber qué cliente hizo la compra y qué videojuego compró. Son claves ajenas o FK.

Paso 3: Diagrama E/R con campos y claves

Para que sea más fácil de copiar, el diagrama se ha colocado en vertical. La entidad COMPRA queda en el centro porque une al CLIENTE con el VIDEOJUEGO.

Paso 4: Explicación de las cardinalidades

  • Un cliente puede realizar muchas compras.
  • Cada compra pertenece a un solo cliente.
  • Un videojuego puede aparecer en muchas compras.
  • Cada compra corresponde a un videojuego concreto.

8. Actividad para el alumnado

Ahora te toca a ti: lee el siguiente enunciado y crea el diagrama Entidad-Relación correspondiente.

Nuevo problema: academia de idiomas

Una pequeña academia de idiomas necesita organizar la información de sus alumnos y de los cursos que ofrece.

Actualmente la academia apunta las matrículas en una hoja de cálculo, pero quiere crear una base de datos para trabajar mejor.

La academia necesita guardar la siguiente información:

  • De cada alumno quiere guardar su nombre, apellidos, teléfono y correo electrónico.
  • De cada curso quiere guardar el nombre del curso, el idioma, el nivel y el precio mensual.
  • Cuando un alumno se matricula en un curso, quiere guardar la fecha de matrícula.
Pista: un alumno puede matricularse en varios cursos y un curso puede tener varios alumnos.
